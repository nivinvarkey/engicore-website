03 — Architecture

Built like the drawings it manages.

Every module follows the same discipline, so the platform scales the way a drawing register does — predictably, one sheet at a time.

§1

Module-per-feature

Each workflow is its own file, with a background worker handling the heavy lifting separate from the tab's display logic. A change to one module can't leak into another.

§2

Shared utilities

Buttons, activity logging, and the current user are pulled from one shared layer, so every tab looks, behaves, and reports the same way by default.

§3

Logged by default

Every completed action writes to a central activity log. Nothing runs silently — if a batch job touched a file, there's a record of it.

§4

Self-updating

New builds ship through the release channel and the desktop app pulls them down on its own. No manual reinstall across 40+ workstations.
